# Plico v3.8 基准测试报告

## 0. 版本说明

v3.8 核心改进：
1. **recall_routed 管线接入**：意图分类 + 7 信号 RFE + MMR 多样性 + HyDE 假设文档
2. **MemoryConsolidationEngine 上线**：会话结束自动执行语义去重（sim≥0.90）、矛盾检测、重要性衰减/增强
3. **IntentSemanticNetwork 学习闭环**：会话结束自动从轨迹学习意图关联（公理 9：越用越好）
4. **ExperienceMiner 真实实现**：基于滑动窗口的重复模式提取，接入 TrajectoryTracker
5. **ContextQualityEngine 补全**：真实 token 统计、CAS 接入、L0 摘要生成
6. **SkillForge 意图-技能匹配**：从硬编码字符串匹配升级为 embedding 余弦相似度

## 4. 根因分析

### v38 提升的核心原因

1. **`recall_routed` 管线**：意图分类将查询路由到最优检索策略
   - temporal 查询 → 时间感知排序
   - multi-hop 查询 → 扩展上下文窗口 + 图遍历
   - factual 查询 → 精确匹配优先

2. **MemoryConsolidationEngine**：会话结束自动去重（sim≥0.90），减少噪声记忆
3. **IntentSemanticNetwork 学习**：从历史轨迹学习意图关联，预加载相关上下文

### 仍存在的问题

| 问题 | 根因 | 影响 |
|------|------|------|
| F1 偏低（0.142） | reader 模型改述答案，F1 衡量 token 重叠而非语义 | LLM Score 3.44 更能反映真实质量 |
| Multi-hop F1 低（0.038） | 需要跨会话链接推理，当前无显式推理链 | LLM Score 2.64 说明答案质量尚可 |
| LME 上下文命中率下降（65%） | 数据集扩大至 500 题，难度增加 | 需要提升检索召回率 |
| Preference 类问题差（LLM 1.20） | 偏好记忆需要特殊的提取和检索策略 | 需要专门的偏好记忆管线 |

## 5. 下一阶段目标（v39）

### 优先级 1：多跳推理管线
- 实现思维链检索：搜索 → 提取实体 → 再次搜索
- 添加图遍历，通过 KG 边关联相关记忆
- 目标：multi-hop LLM Score ≥ 3.0（当前 2.64）

### 优先级 2：检索召回率提升
- 优化 `recall_routed` 的 HyDE 策略
- 添加查询扩展（同义词、上下位词）
- 目标：LME 上下文命中率 ≥ 0.80（当前 0.65）

### 优先级 3：偏好记忆管线
- 实现偏好提取器：从对话中识别用户偏好
- 添加偏好记忆专用检索策略
- 目标：preference LLM Score ≥ 2.5（当前 1.20）

### 优先级 4：答案质量优化
- 添加答案提取 prompt，强制生成简洁回答
- 实现答案归一化后再计算 F1
- 目标：LoCoMo F1 ≥ 0.20（当前 0.142）
