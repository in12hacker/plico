# Plico v0.6 里程碑计划（草案）：Dogfood 闭环 + 执行硬化 + FS 健康

> **Tier B** — 在 **v0.5 交付之后** 执行的下一迭代；与 [v0.5_intent_router_agent_enforcement.md](v0.5_intent_router_agent_enforcement.md) 的「落地对照」衔接。  
> **命名**：v0.6 为建议号；若范围更小可标为 v0.5.1，但本文件以 v0.6 为工作标题。

---

## 1. 里程碑目标

1. **Dogfood 可重复**：同一 `PLICO_ROOT` + `agent_id`（如 `plico-dev`）下，语义对象的 **增 / 查 / 读 / 改 / 删** 有自动化验收，不再依赖「口头验证」。
2. **计划与工具链一致**：`plico-dogfood` SKILL 中的示例命令与 `aicli` 行为一致（含 `--agent`）。
3. **v0.5 未闭合项收口**：`cpu_time_quota` 在 intent 执行路径上可观测；Clippy 策略与仓库现状对齐或分阶段清零。
4. **代码健康（延续 v0.5 Task 4）**：`semantic_fs.rs` / `graph.rs` 机械拆分（独立 PR，避免与 Dogfood 测试绞在一起）。

## 3. 任务分解

### Task A — Dogfood 语义 CRUD 自动化（P0）

在 `tests/cli_test.rs`（或新建 `tests/dogfood_cli_test.rs`）中，使用 **临时目录** + `EMBEDDING_BACKEND=stub`，固定 `AGENT=plico-dev`（或任意非 `cli` 字符串）：

| 步骤 | CLI / 行为 | 验收要点 |
|------|------------|----------|
| 增 | `put --content … --tags … --agent plico-dev` | 成功返回 CID |
| 查 | `search --query … --require-tags … --agent plico-dev` | 结果含上述 CID |
| 读 | `get <CID> --agent plico-dev` | 正文与 tag 一致 |
| 改 | `update --cid … --content … --agent plico-dev` | 成功；`get` 可见新内容（注意 CID 是否变更以实际 API 为准） |
| 删 | `delete --cid … --agent plico-dev` | 若默认策略拒绝 Delete：本任务 **明确** 验收路径——先 `grant`（若 CLI 暴露）或通过 **所有者 + 文档化权限** 完成删除负例/正例各一 |

**非目标**：不强制使用真实 `/tmp/plico-dogfood` 路径跑 CI（避免本机污染）；以 `tempdir` 为准即可等价验证行为。

### Task B — SKILL 与文档（P1）

- 更新 `~/.claude/skills/plico-dogfood/SKILL.md`（或仓库内若未来迁入的副本）：`get`/`update`/`delete` 示例一律带 **`--agent $AGENT`**，与 v0.5 计划写入身份一致。
- 本文与 [dogfood_cli_read_agent_gap.md](dogfood_cli_read_agent_gap.md) 互链；**状态表**标明 Task A 完成后可关闭「整链未验」风险。

### Task C — `cpu_time_quota` 执行硬化（P1）

- 在 `KernelExecutor` / `dispatch` 路径对单次 intent 执行包 `tokio::time::timeout`（`0` = 不限）。
- 集成测试：极小 quota 下 intent **失败或可观测错误**，且 agent 不误杀为 Terminated（与 v0.5 计划表述一致）。

### Task D — Clippy 策略（P1）

- 二选一并在 `AGENTS.md` **Clippy Policy** 中写死：  
  - **A**：CI / 贡献指南以 `cargo clippy -- -D warnings` 为门槛，分 PR 清零现存告警；或  
  - **B**：明确「当前允许告警清单 + 渐进清零」，避免文档与真实 `cargo clippy` 长期矛盾。

### Task E — `semantic_fs` / `graph` 拆分（P2）

- 按 v0.5 计划表机械拆分；每子 PR 单文件主题；更新 `src/fs/INDEX.md`。

## 4. 验收标准（本里程碑完成定义）

- [ ] 存在至少 **1** 个集成测试：`plico-dev`（或等价固定 agent）完成 **put → search → get → update →（delete 正例或文档化负例+grant 正例）**。
- [ ] Dogfood SKILL 中读取/更新/删除示例与实现一致。
- [ ] `cpu_time_quota > 0` 时 intent 执行可被超时打断（或有明确 `SchedulerError`），并有测试覆盖。
- [ ] `AGENTS.md` 中 Clippy 描述与仓库策略一致。
- [ ] `cargo test` 全绿；若选 A，则 `cargo clippy` 无告警（或分阶段里程碑内声明例外）。

---

## 5. 依赖与风险

- **权限**：Delete 可能需要显式 grant；测试设计前读 `PermissionGuard` 与 `semantic_delete` 契约。
- **Update 是否改 CID**：测试断言须与 `semantic_update` 实际返回值一致。
