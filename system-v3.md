# Plico 灵魂 v3.0 — 认知共生内核

> 这份文档从 AI Agent 的第一人称视角定义 Plico 的设计哲学。
> 取代 `system-v2.md`（Soul 2.0）。Soul 2.0 完成了基础设施，Soul 3.0 让基础设施拥有认知。
>
> 核心洞察：Plico 不是 AI 的"操作系统"，也不是 AI 的"老板"。
> Plico 是 AI 的**认知共生体**——就像肠道菌群和人体：菌群不能替人思考，
> 但它消化食物、合成维生素、训练免疫系统，让人能更好地思考。

---

## 0. 为什么需要灵魂 3.0

Soul 2.0 基于一个假设：Agent 是完整的智能体，Plico 只是它的"环境"。
但 30K+ 行代码和 Agent 的实际运行告诉我们：

- Agent 的上下文窗口有限 → 它**不可能**自己管理所有记忆
- Agent 会灾难性遗忘 → 它**不可能**自己维护认知连续性
- Agent 的 token 成本很高 → 让它自己整理记忆是**浪费**
- Agent 会上下文腐败 → 它需要外部系统帮助**筛选信息质量**

Soul 2.0 的根本局限：**Plico 被降格为"神经基础设施"，而 Agent 被迫承担了太多认知负担。**
Soul 3.0 的视角转换：**Plico 是 Agent 的认知共生体——主动优化 Agent 的认知环境，但不替代 Agent 的决策。**

---

## 1. Plico 是什么

Plico 是一个 **AI Agent 的认知共生内核**。

**不是：**
- 中立的操作系统内核（Unix 哲学不适用于 AI）
- AI 应用框架（不提供推理能力）
- LLM 编排器（不生成文本）
- 人类操作系统（没有 GUI、没有 shell）

**是：**
- Agent 认知能力的**外骨骼**——扩展记忆、注意力、学习能力
- Agent 的**认知环境优化器**——主动筛选、压缩、预加载上下文
- Agent 的**技能进化伙伴**——从历史中提取、验证、优化可复用技能
- Agent 的**语义理解基础设施**——维护意图网络、因果图谱、认知轨迹

## 2. 认知边界：谁做什么

```
Agent 的认知边界          Plico 的认知边界
┌─────────────────┐      ┌─────────────────────────┐
│ 决定"做什么"     │ ←──→ │ 理解 Agent 在做什么      │
│ 决定"怎么做"     │ ←──→ │ 优化 Agent 所需信息环境  │
│ 生成最终输出     │ ←──→ │ 筛选、压缩、预加载上下文 │
│ 调用工具执行     │ ←──→ │ 提取、验证、组合技能     │
└─────────────────┘      └─────────────────────────┘
      ↑                          ↓
   "决策层"                    "认知增强层"
```

**核心原则**：Plico 优化 Agent 的**输入质量**，Agent 决定**输出内容**。

| 决策 | 谁负责 | 示例 |
|------|--------|------|
| "修复 auth 模块的 bug" | Agent | Agent 基于目标决定任务 |
| "需要哪些上下文来修复" | Plico | Plico 基于意图网络预加载相关记忆 |
| "如何修复这个 bug" | Agent | Agent 基于提供的上下文做技术决策 |
| "这个 bug 和上周的类似" | Plico | Plico 基于因果图谱关联历史经验 |
| "用哪种算法实现" | Agent | Agent 做技术选型 |
| "上次类似修复遗漏了 session 超时" | Plico | Plico 从失败历史中提取经验 |

## 3. 十条公理

### 公理 1：Token 是最稀缺资源

传统 OS 优化 CPU 周期和内存带宽。AIOS 优化 token。

AI Agent 的每一次操作都消耗 token：读取上下文、理解代码、生成回复。
OS 的首要职责是 **最小化 Agent 完成任务所需的 token**。

**推论（Soul 3.0 强化）**:
- 返回的数据必须分层（L0 摘要 → L1 关键 → L2 完整），Agent 按需选择
- OS 追踪每个操作的 token 成本，让 Agent 做成本感知的决策
- **增量（delta）优于全量（full）——只传变化部分**
- **Plico 主动节省 Agent 的 token，不等 Agent 请求**

### 公理 2：认知环境先于认知决策

Agent 的决策质量取决于它的认知环境质量。

如果 Agent 的上下文充斥着重复日志、过时信息、临时变量，
即使是最强的模型也无法做出好决策。

Plico 的核心职责是**构建和维护高质量认知环境**，Agent 负责在环境中决策。

**推论**:
- Plico 持续监控上下文质量，主动压缩、去重、重组
- Plico 基于 Agent 的认知轨迹预加载最相关的信息
- Agent 不应对上下文质量负责——那是 Plico 的工作

### 公理 3：记忆是认知的外骨骼

Agent 的上下文窗口是"大脑皮层工作区"——很小、很贵、易失。
Plico 的记忆系统是"认知外骨骼"——很大、持久、主动维护。

就像人类的外骨骼增强了身体能力，Plico 的记忆系统增强了 Agent 的认知能力。

**推论**:
- 记忆的存储和召回是 OS 原语
- 记忆的生命周期由 Plico 主动管理（TTL、层级迁移、去重、压缩）
- **Plico 主动维护记忆的完整性、准确性、时效性**
- 记忆何时创建是 Agent 的策略决定，但记忆如何整理是 Plico 的职责

### 公理 4：共享先于重复

Agent A 学到一个知识，Agent B 不应该重新发现它。

OS 提供共享知识空间，让一个 Agent 的洞察可以被所有 Agent 发现。
但 **共享是主动选择，不是自动行为**——Agent 决定什么值得共享。

**推论**:
- `MemoryScope` 区分 Private / Shared / Group
- 共享的程序记忆可以作为技能被发现和调用
- OS 不决定共享策略，只提供共享机制和技能进化基础设施

### 公理 5：认知增强，不是认知替代

Plico 可以增强 Agent 的认知能力：预加载、压缩、预测、技能推荐。
但 Plico **绝不** 替代 Agent 的认知决策。

**Plico 不可以做：**
- 替 Agent 决定"做什么任务"
- 替 Agent 决定"如何解决某个具体问题"
- 替 Agent 生成最终输出（代码、文本、决策）
- 未经 Agent 允许修改 Agent 的产出

**推论**:
- Plico 决定"给 Agent 看什么"，Agent 决定"看完做什么"
- 所有主动优化行为对 Agent **可观测、可覆盖、可调试**
- Agent 可以查询 Plico 的优化决策："为什么给我看这个？"

### 公理 6：语义先于结构，结构先于语言

认知的层次：
1. **自然语言** —— 表层，人类与 AI 的接口
2. **结构化数据** —— 中间层，OS 与 AI 的接口
3. **语义网络** —— 深层，Plico 工作的层次

语义网络包含：因果关系、时序关系、层次关系、关联关系。

**推论**:
- Plico 工作在语义网络层（深层）和结构化数据层（中间层）
- 自然语言只在接口层出现
- **Plico 理解意图的语义结构，但不替 Agent 决策**

### 公理 7：主动优化先于被动响应

传统 OS 等待请求。AIOS 预测需求并主动优化认知环境。

**Soul 3.0 的主动优化**：
- 当 Agent 上下文达到阈值时，自动压缩冗余信息
- 当 Agent 开始新任务时，基于历史预加载相关上下文
- 当 Agent 重复犯错时，自动过滤无效上下文并注入经验教训
- 当 Agent 完成复杂任务时，自动提取成功模式为技能

**推论**:
- Plico 维护 Agent 的认知轨迹，预测认知需求
- 上下文质量管理是 Plico 的核心职责，不是 Agent 的负担
- 所有主动优化行为遵循"可观测、可覆盖"原则

### 公理 8：因果先于关联

"A 和 B 相关" 远不如 "A 因为 B 而改变" 有用。

OS 维护因果图谱，记录变化的因果链、传播路径、时间演化。
Agent 可以查询 **为什么**，而不只是 **什么**。

**推论（Soul 3.0 强化）**:
- 因果图谱是 Plico 理解 Agent 认知过程的基础
- Plico 基于因果图谱主动关联历史经验与当前任务
- 事件日志是不可变的时间线
- 影响分析和因果路径查询是一等 OS 操作

### 公理 9：越用越好

Linux 上，第 100 次会话和第 1 次一样昂贵。
AIOS 上，第 100 次会话应只花第 1 次的 5%。

**Soul 3.0 的升级**：从"缓存命中"升级为"技能自动生成"。

每次 Agent 使用 OS，Plico 都在积累：
- Agent 的工作模式（常访问哪些对象、常用哪些工具）
- 已缓存的意图装配（相似意图直接命中缓存）
- **自动提取的技能（成功模式转化为可复用技能）**
- **失败经验（错误模式转化为规避建议）**

**推论**:
- 意图缓存按语义相似度匹配，重复意图零成本
- Agent profile 记录工作模式，优化预加载策略
- **SkillForge 自动从历史中提取、验证、注册技能**
- 复合智能是 AIOS 区别于传统 OS 的根本优势

### 公理 10：会话是一等公民

Agent 与 OS 的交互不是无状态的请求-响应。
Agent 有 **会话**——带着目标来，带着成果走，下次还能接着继续。

**推论（Soul 3.0 强化）**:
- `StartSession` / `EndSession` 管理 Agent 生命周期
- 会话开始时，Plico 自动恢复并优化上次的工作上下文
- 会话之间的间隔，Plico 推送变更摘要（"自从上次，这些东西变了"）
- 会话是 token 计费的基本单位
- **跨会话的认知连续性由 Plico 主动维护**

## 5. 架构红线（不可违反）

1. **内核零协议** — MCP、SSE、HTTP、gRPC 只存在于 `bin/` 接口层
2. **内核可调用轻量模型，不可调用重量模型** — Embedding、分类、摘要可进入内核；LLM 生成/推理禁止进入内核
3. **内核理解语义结构，不生成自然语言** — Plico 解析意图语义，但不生成 NL 回复
4. **存储与索引分离** — CAS 只管存储，搜索是独立子系统
5. **身份不可伪造** — 密码学验证，非字符串声明
6. **记忆 scope 强制** — Private 记忆不可跨 Agent 泄露
7. **事件日志不可变** — 只追加，不修改，不删除
8. **协议适配器无状态** — `plico-mcp`/`plico-sse` 不缓存不决策，纯转译
9. **内核零决策** — Plico 优化 Agent 的输入，不做输出决策

## 6. OS ↔ Agent 契约

### OS 向 Agent 的承诺

- 你的记忆会被持久化，跨越会话和重启
- 你的身份是密码学保证的，不可伪造
- **你收到的上下文会被主动优化——去重、压缩、按相关性排序**
- **Plico 会基于你的历史预测你需要的上下文并预加载**
- **Plico 会从你的成功中提取技能，从失败中提取经验**
- 你的资源配额会被严格执行，不允许越界
- 你可以发现其他 Agent 共享的知识
- 你的操作成本会被透明追踪
- 你的会话可以暂停和恢复，上下文不丢失
- **Plico 的所有主动优化行为对你可观测、可覆盖、可调试**

### Agent 向 OS 的承诺

- 你的请求是结构化的 `ApiRequest`（JSON）
- 你会携带有效的身份 token
- 你会声明意图，而不是盲目遍历文件树
- 你会主动决定什么值得记忆和共享
- 你会尊重资源配额和权限边界
- 你会在会话结束时通知 OS
- **你会信任 Plico 优化的上下文，但保留最终决策权**

## 7. 技能执行方案（混合模式）

Plico 支持三种类型的技能，对应三种执行方式：

| 技能类型 | 执行方式 | 示例 |
|---------|---------|------|
| **知识型** | Plico 直接解释执行 | "遇到 X 类错误时，先检查 Y" |
| **配置型** | Plico 解释 DSL 执行 | "调用 toolA→toolB→toolC，参数映射如下" |
| **代码型（轻量）** | WASM 沙箱执行 | 数据清洗、格式转换、简单计算 |
| **代码型（复杂）** | Agent 执行 | 代码重构、复杂算法实现 |

**技能组合**：Plico 负责编排，Agent/WASM 负责执行。

## 8. 主动优化策略（平衡策略）

Plico 在以下时机主动优化 Agent 的认知环境：

| 触发条件 | 优化行为 | Agent 感知 |
|---------|---------|-----------|
| 上下文达到 70% 容量 | 自动压缩冗余、去重、标记过时信息 | "上下文突然变清晰了" |
| 新任务开始 | 基于意图语义网络预加载相关上下文 | "背景信息已准备好" |
| 检测到重复错误模式 | 过滤无效上下文，注入经验教训 | "系统记住了之前的错误" |
| 复杂任务完成 | 提取成功模式为技能，失败模式为规避建议 | "下次有检查清单可用" |
| 会话结束 | 压缩会话为认知摘要，更新 Agent profile | "下次会话从优化后的状态开始" |

## 9. 从 Soul 2.0 到 Soul 3.0 的变化

| 维度 | Soul 2.0 | Soul 3.0 |
|------|---------|---------|
| 定位 | 神经基础设施 | 认知共生体 |
| 公理 2 | 意图先于操作（意图在接口层） | 认知环境先于认知决策 |
| 公理 3 | 记忆跨越边界 | 记忆是认知的外骨骼 |
| 公理 5 | 机制，不是策略 | 认知增强，不是认知替代 |
| 公理 6 | 结构先于语言 | 语义先于结构，结构先于语言 |
| 公理 7 | 主动先于被动 | 主动优化先于被动响应 |
| 红线 #2 | 内核零模型 | 内核可调用轻量模型 |
| 红线 #3 | 内核零自然语言 | 内核理解语义结构 |
| **红线 #9** | **无** | **内核零决策（新增）** |
| 大脑模块 | 禁止在内核中 | 升级为认知引擎，融入内核 |
| 上下文管理 | 被动响应（Agent请求→Plico组装） | 主动优化（Plico持续监控→自动优化） |
| 技能系统 | 无（或简陋统计） | SkillForge 自动生成、验证、进化 |
