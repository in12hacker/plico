Bridge the two biggest "soul gaps" between system.md and current implementation:
1. **Natural Language → Structured Intent** translation (NUI layer) — system.md says "自然语言是主要接口"

## Task 1: Intent Router — NL → ApiRequest Translation

**Module**: `src/intent/`

A thin NL-to-structured-intent translation layer. An agent sends free-form text describing what it wants to do; the system returns one or more `ApiRequest` or `ToolCall` actions.

### Types

```rust
// src/intent/mod.rs

pub trait IntentRouter: Send + Sync {
    fn resolve(&self, text: &str, agent_id: &str) -> Result<Vec<ResolvedIntent>, IntentError>;
}

pub struct ResolvedIntent {
    pub confidence: f32,           // 0.0–1.0
    pub action: ApiRequest,        // the structured request
    pub explanation: String,       // why this action was chosen
}

pub enum IntentError {
    Ambiguous(Vec<ResolvedIntent>),  // multiple candidates, agent should clarify
    Unresolvable(String),             // no matching action found
    LlmUnavailable(String),          // LLM backend failed
}
```

### Implementations

1. **HeuristicRouter** (`src/intent/heuristic.rs`):
   - Pattern matching for common operations
   - Patterns: "find/search/查找" → Search, "store/save/保存" → Create, "delete/删除" → Delete, etc.
   - Temporal NL phrases delegated to existing TemporalResolver
   - No external dependency — always available

2. **LlmRouter** (`src/intent/llm.rs`):
   - Sends NL + tool catalog to configured LLM (Ollama)
   - System prompt includes ToolRegistry.list() descriptions + ApiRequest schema
   - Parses LLM JSON output into ResolvedIntent
   - Optional — requires Ollama

3. **ChainRouter** (`src/intent/mod.rs`):
   - Default: tries Heuristic first, falls back to LLM if confidence < threshold
   - Configurable threshold (default 0.7)

### API Integration

```rust
// New ApiRequest variant
ApiRequest::IntentResolve { text: String }

// Response: ApiResponse with resolved_intents field
pub resolved_intents: Option<Vec<ResolvedIntent>>
```

### CLI

```bash
aicli intent "find documents about agent scheduling from last week"
# Output: Resolved 2 actions:
#   1. [0.92] Search { query: "agent scheduling", temporal: "last week" }
#   2. [0.45] KGExplore { start: "agent-scheduler", depth: 2 }
```

### Files
- Create: `src/intent/mod.rs`, `src/intent/heuristic.rs`, `src/intent/llm.rs`
- Modify: `src/lib.rs`, `src/api/semantic.rs`, `src/kernel/mod.rs`, `src/bin/aicli.rs`

### Acceptance Criteria
- [ ] HeuristicRouter resolves at least 8 common patterns correctly
- [ ] LlmRouter (when Ollama available) resolves free-form queries
- [ ] ChainRouter fallback works: heuristic → LLM → error
- [ ] `aicli intent "..."` produces formatted output
- [ ] Temporal phrases ("last week", "yesterday") correctly delegate to TemporalResolver

## Task 2: Agent Resource Enforcement

Wire the existing `AgentResources` struct into actual runtime enforcement.

### Current State
```rust
// src/scheduler/agent.rs — EXISTS but unused
pub struct AgentResources {
    pub memory_quota: usize,       // max memory entries
    pub cpu_time_quota: u64,       // max ms per intent execution
    pub allowed_tools: Vec<String>, // empty = all allowed
}
```

### Changes

1. **Memory Quota Enforcement** (`src/memory/layered.rs`):
   - `store()` checks agent's memory_quota before inserting
   - Returns `MemoryError::QuotaExceeded { agent_id, current, limit }`

2. **CPU Time Enforcement** (`src/scheduler/dispatch.rs`):
   - `KernelExecutor` wraps intent execution in `tokio::time::timeout`
   - Timeout duration = agent's `cpu_time_quota` (0 = unlimited)
   - On timeout: intent marked Failed, agent state → Ready (not terminated)

3. **Tool Allowlist** (`src/kernel/mod.rs`):
   - `execute_tool()` checks agent's `allowed_tools` before dispatch
   - Empty list = all tools allowed (backward compatible)
   - Blocked tools return `ToolResult::error("tool not in agent's allowed list")`

4. **Resource Management API**:
   ```rust
   ApiRequest::AgentSetResources { agent_id: String, resources: AgentResources }
   ```
   - Permission: only agent owner or trusted agents can modify
   - CLI: `aicli agent set-resources <agent-id> --memory-quota 100 --cpu-time-quota 5000 --allowed-tools "cas.get,cas.put"`

### Files
- Modify: `src/scheduler/dispatch.rs`, `src/scheduler/agent.rs`, `src/memory/layered.rs`, `src/kernel/mod.rs`, `src/api/semantic.rs`, `src/api/permission.rs`, `src/bin/aicli.rs`

### Acceptance Criteria
- [ ] Agent with memory_quota=10 gets QuotaExceeded on 11th store
- [ ] Agent with cpu_time_quota=100 is killed after 100ms
- [ ] Agent with allowed_tools=["cas.get"] gets error on "memory.store"
- [ ] `aicli agent set-resources` works end-to-end
- [ ] Backward compatible: existing agents (quota=0) are unrestricted

## Task 3: Agent Messaging Protocol

Enable kernel-mediated agent-to-agent communication.

### Types

```rust
// src/scheduler/messaging.rs
pub struct AgentMessage {
    pub id: String,
    pub from: String,
    pub to: String,
    pub payload: serde_json::Value,
    pub timestamp_ms: u64,
    pub read: bool,
}

pub struct AgentMailbox {
    messages: Vec<AgentMessage>,
    capacity: usize,  // default 100
}
```

### API

```rust
ApiRequest::SendMessage { from: String, to: String, payload: Value }
ApiRequest::ReadMessages { agent_id: String, unread_only: bool }
ApiRequest::AckMessage { agent_id: String, message_id: String }
```

### Permission
- Sender must have `PermissionAction::SendMessage` grant for target agent
- Reading messages: agent can only read its own mailbox
- Trusted agents can read any mailbox

### CLI
```bash
aicli agent send <from-id> <to-id> --payload '{"task":"summarize","cid":"abc123"}'
aicli agent messages <agent-id> [--unread]
aicli agent ack <agent-id> <message-id>
```

### Files
- Create: `src/scheduler/messaging.rs`
- Modify: `src/scheduler/mod.rs`, `src/scheduler/agent.rs`, `src/kernel/mod.rs`, `src/api/semantic.rs`, `src/api/permission.rs`, `src/bin/aicli.rs`

### Acceptance Criteria
- [ ] Agent A sends message to Agent B; B reads unread messages
- [ ] Unauthorized sender gets permission denied
- [ ] Ack marks message as read
- [ ] Mailbox capacity enforced (oldest dropped when full)
